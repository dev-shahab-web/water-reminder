# Support

This starter is maintained as a reusable framework foundation.

## Questions

Use GitHub Discussions if enabled. Otherwise, open an issue with the `question` label.

## Bugs

Use the bug report template and include reproduction steps, environment details, and verification commands.

## Feature Requests

Use the feature request template. Architecture-impacting requests should include alternatives considered.

## Security

Security issues should follow [SECURITY.md](./SECURITY.md).
