# AI Collaboration

## Purpose

This repository is designed to be maintainable by humans and AI agents.

## Rules for AI Agents

- Read `PROJECT_CONTEXT.md` before architecture work.
- Read relevant docs before implementation.
- Do not invent architecture when docs define it.
- Do not implement multiple unrelated systems in one step.
- Do not cross documented dependency boundaries.
- Do not add dependencies without documenting why.
- Verify every step with the project quality gate.
- Stop after each major milestone for review.

## Documentation First

If architecture is unclear, document the decision before implementing code. The docs are part of the framework, not a side artifact.

## Review Behavior

AI should surface risks, tradeoffs, and alternatives when architecture changes are proposed.
