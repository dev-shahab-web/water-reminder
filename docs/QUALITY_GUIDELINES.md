# Quality Guidelines

## Quality Gate

Every meaningful change should satisfy:

- TypeScript passes.
- ESLint passes.
- Tests pass or missing tests are justified.
- Formatting is clean.
- Documentation is updated when behavior or architecture changes.
- Dependency boundaries are respected.

## Definition of Quality

Quality means the code can be safely reused across multiple apps, understood by future maintainers, and extended without accidental coupling.

## Review Focus

Reviews should prioritize correctness, architecture boundaries, type safety, accessibility, performance risks, and missing tests.
