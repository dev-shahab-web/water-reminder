# Code Review

## Review Priorities

1. Correctness
2. Architecture boundaries
3. Type safety
4. Error handling
5. Test value
6. Accessibility
7. Performance
8. Maintainability

## Reviewer Questions

- Does this change belong in the current milestone?
- Does it preserve dependency direction?
- Is the abstraction justified?
- Are errors handled at the right boundary?
- Are tests meaningful?
- Could this be reused across future apps?
- Are docs updated when needed?
